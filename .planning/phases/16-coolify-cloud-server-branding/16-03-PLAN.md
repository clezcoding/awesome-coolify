<assumption_delta_decision>
**Decision type:** add-alongside (NOT promote).
**Primary noun:** MCP serverInfo (one metadata object emitted at initialize).
**Resolution:** icons and title/description/websiteUrl are first-class peer fields in one serverInfo object — not a second identity model. The SDK `name`/`version` stay as the canonical identity; `title`/`description`/`websiteUrl`/`icons` are complementary fields per MCP SDK ImplementationSchema. No promotion of icons above name; no demotion of title/description below icons.
**Rationale:** The @modelcontextprotocol/server ImplementationSchema extends BaseMetadataSchema (name, title) with IconsSchema (icons) plus version, websiteUrl, description — all peer fields in one object. Adding icons alongside title/description/websiteUrl matches the SDK contract; no field is promoted or demoted.
</assumption_delta_decision>
