<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → stdio server | Untrusted tool args; validated by Zod in later plan; health has no user-supplied secrets |
| Server → Coolify API | Bearer token crosses network; TLS verification configurable |
| Env vars → process | COOLIFY_TOKEN loaded from mcp.json env; must not leak to stdout |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Spoofing | Coolify API | medium | mitigate | Bearer token in Authorization header per D-22 |
| T-01-02 | Information Disclosure | logs/responses | high | mitigate | Health response returns host only per D-21; no token in content |
| T-01-03 | Information Disclosure | error messages | high | mitigate | Minimal client errors; full redaction in plan 01-04 |
| T-01-04 | Tampering | env input | high | mitigate | Zod url() + min(1) validation in loadEnv per D-02 |
| T-01-05 | Information Disclosure | TLS | medium | mitigate | COOLIFY_VERIFY_SSL default true; false only when explicitly set |
| T-01-SC | Tampering | npm install | high | mitigate | Human checkpoint Task 1 for [SUS] @modelcontextprotocol/server before install |

</threat_model>
