## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → prompts/list, prompts/get | Untrusted agent/client requests prompt guidance; handlers return static-ish text built from args |
| MCP client → tool description surface | Description strings are static metadata served to the host IDE; no dynamic user input flows into them |
| Prompt handler → filesystem | Forbidden: handlers must not read .coolify/manifest.json or any other file from disk (D-10) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-19-04 | Information Disclosure | prompt message bodies | high | mitigate | Prompt handlers compose text from caller-supplied args (uuid, name, server_uuid, force) only; never echo credentials, tokens, env values, or response data; soft Note steers agent to manifest/ask-user without reading the manifest from disk (D-10) |
| T-19-05 | Tampering | prompt args (uuid, force, name, server_uuid, project_uuid) | medium | mitigate | All prompt args are optional strings validated by z.string().optional() argsSchema; handlers treat them as opaque text and never eval or exec them; force parsed via strict `=== 'true'` check |
| T-19-06 | Information Disclosure | tool description strings | low | accept | Descriptions are static metadata composed from co-located constants; no user input flows into them; catalog/footer strings are hand-maintained and reviewed |
| T-19-07 | Denial of Service | prompt handler invocation | low | accept | Prompt handlers return a fixed-size messages array (~2 messages, ≤~600 words); no loops, no recursion, no I/O — bounded by construction |
| T-19-SC | Tampering | npm package installs | low | accept | No new packages installed in this plan — reuses zod and @modelcontextprotocol/server already pinned; [SUS] flag accepted per RESEARCH |
</threat_model>
