    - Skips blank lines and lines starting with # (after optional leading whitespace)
    - Strips surrounding single quotes (no escape processing)
    - Strips surrounding double quotes and unescapes \\n, \\", \\\\ inside
    - Preserves empty values (KEY=)
    - Preserves leading/trailing whitespace inside double quotes
    - diffEnvs(local: ParsedEnv[], remote: Env[]) returns { added, updated, unchanged, removed } keyed by env key
    - added = keys in local not in remote
    - updated = keys in both where local.value !== remote.value
    - unchanged = keys in both where values match
    - removed = keys in remote not in local
    - detectConflicts(local, remote, policy: 'overwrite'|'keep_remote'|'abort') returns Conflict[] where a conflict is a key whose remote value differs from the local snapshot baseline (out-of-band edit)
    - With policy 'overwrite' conflicts is always empty (caller will clobber remote)
    - With policy 'keep_remote' conflicts are returned; caller treats them as keep-remote decisions (no throw)
    - With policy 'abort' conflicts are returned; caller treats them as abort decisions (no throw)
    - detectConflicts does NOT accept 'fail' or 'skip' (D-08 enum is overwrite|keep_remote|abort only)
    - All three functions are pure (no IO, no side effects, no network)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Handler → client.ts | Handler passes user-supplied uuid, env_uuid, payload; client constructs URL and body |
| .env content → env-parser.ts | Untrusted local file content crosses into parser; must not execute or escape |
| client.ts → Coolify API | Outbound HTTP with bearer token; errors mapped back through mapApiError |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-12-01-01 | Information Disclosure | error messages from client.ts | high | mitigate | mapApiError already redacts secrets via redactSecrets; do not append raw response bodies to error envelopes. Verify RECOVERY_HINTS does not echo env values. |
| T-12-01-02 | Tampering | createEnv database is_preview guard | high | mitigate | Client-side guard throws COOLIFY_VALIDATION_ERROR before HTTP when is_preview is true on database calls; OpenAPI also rejects but we fail fast to avoid leaking the payload. |
| T-12-01-03 | Tampering | updateEnvViaBulk database is_preview guard | high | mitigate | Same guard on every entry in the bulk array; throw on first offending entry. |
| T-12-01-04 | Denial of Service | parseEnvFile on huge .env content | low | accept | Pure string processing; caller responsible for size limits. Document a soft limit in JSDoc (e.g. 1 MiB). |
| T-12-01-05 | Information Disclosure | env-parser.ts error paths | low | mitigate | Parser throws on malformed input without echoing the offending value; error message includes only the line number and key name. |
| T-12-01-06 | Tampering | npm/pip/cargo installs | high | mitigate | No new packages installed in this plan (ofetch, zod, yaml already audited [OK] in 12-RESEARCH.md). No slopcheck checkpoint needed. |
