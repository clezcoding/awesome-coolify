<objective>
Wire the 7 read/CRUD domain tools (resource, system, diagnose, server, private_key, project, environment) to accept an optional `instance` routing param and resolve credentials per-request via InstanceManager. This completes CTX-06 alongside Plan 15-03 (which covers the 5 lifecycle tools). Both plans run in Wave 2 parallel — no file overlap. meta and docs tools are intentionally excluded because they make no Coolify API call.

Purpose: Without per-request credential resolution, every read/CRUD call hits the single env-configured instance — making multi-instance routing impossible. Read tools are lower-risk than mutation tools (Plan 15-03) but still must not leak across instances.
Output: 7 tool files with schema + handler updates; integration test coverage extended.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| agent → tool args (instance param) | Untrusted slug crosses here; unknown names must not silently fall back |
| handler → Coolify API | Resolved creds cross here; per-request client prevents cross-instance leakage |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Status | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|--------|-------------|-----------------|
| T-15-04 | Spoofing | unknown instance param silently falling back to default | high | mitigated | mitigate | resolveCredentials throws COOLIFY_INSTANCE_NOT_FOUND for unknown names (D-12); handlers wrap via wrapMcpError |
| T-15-05 | Spoofing | partial env mixing URL from env with token from registry | high | mitigated | mitigate | resolveCredentials throws COOLIFY_PARTIAL_ENV (D-13) |
| T-15-12 | Information Disclosure | cross-instance leakage via global mutable client | high | mitigated | mitigate | Each handler constructs a fresh createCoolifyClient per request — no global active client (Pitfall 1) |
| T-15-13 | Information Disclosure | resolved credentials echoed in error messages | medium | mitigated | mitigate | wrapMcpError redacts secrets; do not embed creds in CoolifyApiError messages |
| T-15-15 | Information Disclosure | read tool returning data from wrong instance | high | mitigated | mitigate | Per-request credential resolution + no silent fallback guarantees the named instance — never a different instance |
| T-15-SC | Tampering | npm installs (none new) | high | mitigated | accept | No new external packages |
</threat_model>
