<objective>
Wire the 5 lifecycle/mutation-heavy domain tools (application, service, database, deployment, emergency) to accept an optional `instance` routing param and resolve credentials per-request via InstanceManager. This is half of CTX-06; Plan 15-04 covers the remaining 7 read/CRUD tools. Both plans run in Wave 2 parallel (no file overlap).

Purpose: Without per-request credential resolution, every tool call hits the single env-configured instance — making multi-instance routing (the core Phase 15 value) impossible. Mutation-heavy tools are the highest-risk surface for cross-instance leakage (Pitfall 1), so they get a focused plan.
Output: 5 tool files with schema + handler updates; integration test (src/mcp/integration.test.ts) GREEN for routing.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| agent → tool args (instance param) | Untrusted slug crosses here; unknown names must not silently fall back |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Status | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|--------|-------------|-----------------|
| T-15-04 | Spoofing | unknown instance param silently falling back to default | high | mitigated | mitigate | resolveCredentials throws COOLIFY_INSTANCE_NOT_FOUND for unknown names (D-12); handlers wrap via wrapMcpError |
| T-15-05 | Spoofing | partial env mixing URL from env with token from registry | high | mitigated | mitigate | resolveCredentials throws COOLIFY_PARTIAL_ENV (D-13) |
| T-15-12 | Information Disclosure | cross-instance leakage via global mutable client | high | mitigated | mitigate | Each handler constructs a fresh createCoolifyClient per request — no global active client (Pitfall 1) |
| T-15-13 | Information Disclosure | resolved credentials echoed in error messages | medium | mitigated | mitigate | wrapMcpError redacts secrets; do not embed creds in CoolifyApiError messages |
| T-15-14 | Denial of Service | mutation executes against wrong instance | high | mitigated | mitigate | Per-request credential resolution + no silent fallback guarantees the named instance (or hard error) — never a different instance |
| T-15-SC | Tampering | npm installs (none new) | high | mitigated | accept | No new external packages |
