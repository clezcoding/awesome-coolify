    2. Create `src/utils/yaml-validator.ts` exporting four pure functions:
       - `encodeCompose(yaml: string): string` — `return Buffer.from(yaml, 'utf8').toString('base64');`
       - `decodeCompose(base64: string): string` — wrap in try/catch; on success return `Buffer.from(base64, 'base64').toString('utf8')`; on error return `''` (defensive — service.get on a malformed stored compose should not crash the MCP)
       - `validateCompose(yaml: string): { ok: true } | { ok: false; error: string }` — if `yaml.trim() === ''` return `{ ok: false, error: 'compose YAML is empty' }`; else try `import { parse } from 'yaml'; parse(yaml)`; on throw return `{ ok: false, error: error.message }`; on success return `{ ok: true }`. Do NOT enforce a full Compose schema (D-07 — light validation only).
       - `projectServiceCompose(raw: Record<string, unknown>): Record<string, unknown>` — D-06 projection. Read `raw.docker_compose_raw`; if it is a non-empty string, return `{ ...raw, compose: decodeCompose(raw.docker_compose_raw as string) }` and then `delete (result as any).docker_compose_raw` (or use a computed key without `docker_compose_raw`). If `docker_compose_raw` is absent, empty, or not a string, return `raw` unchanged. Do NOT mutate the input — always return a new object when adding `compose`.

    3. Create `src/utils/yaml-validator.test.ts` with unit tests:

       - encodeCompose: empty string round-trips to empty string
       - decodeCompose: invalid base64 returns empty string (no throw)
       - validateCompose: valid YAML returns `{ ok: true }`
       - validateCompose: empty string returns `{ ok: false, error }`
       - validateCompose: malformed YAML (e.g. `: : :`) returns `{ ok: false, error }`
       - validateCompose: a realistic docker-compose YAML returns `{ ok: true }`
       - projectServiceCompose: input with `docker_compose_raw` (base64 of `'services:\n  redis:\n    image: redis:7'`) -> output has `compose` set to the decoded YAML and `docker_compose_raw` removed
       - projectServiceCompose: input without `docker_compose_raw` -> returned unchanged (same reference, no `compose` field added)
       - projectServiceCompose: input with empty `docker_compose_raw: ''` -> returned unchanged
       - projectServiceCompose: does NOT mutate the input object

    4. Confirm `npm run build` is green (the new import resolves after `npm install`).
