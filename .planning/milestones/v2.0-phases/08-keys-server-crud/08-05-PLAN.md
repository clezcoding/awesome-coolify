<objective>
Close the single diagnosed UAT gap from 08-UAT.md: `private_key.list + reveal:true` is currently rejected at the MCP host JSON Schema layer (Unrecognized key) before the handler runs, instead of returning the structured `COOLIFY_422` envelope that the unit tests and D-11 spec require. Align the live MCP error path with the unit-test path.

Purpose: D-11 intentionally omitted `reveal` from `listActionSchema` and used `.strict()` to enforce the rejection. Unit tests call `handlePrivateKeyAction` directly → Zod `unrecognized_keys` → `throwValidationError` → `COOLIFY_422` envelope. Live Cursor/MCP validates the JSON Schema (`additionalProperties:false`, no `reveal`) before the handler runs, so the call never reaches `parsePrivateKeyAction` and the host returns a generic "Unrecognized key" error instead of the structured `COOLIFY_422`. Security intent (no PEM on list) is met either way; only the error envelope shape diverges. This plan makes the live path match the unit-test path.

Output: Updated `src/mcp/tools/private_key.ts` (list schema accepts `reveal`; handler rejects `reveal:true` with explicit `COOLIFY_422`), updated `src/mcp/tools/private_key.test.ts` (regression test for the handler-level rejection path and for `reveal:false` normal flow), and a one-line description tweak in `src/mcp/server.ts` documenting the dual-layer rejection.
