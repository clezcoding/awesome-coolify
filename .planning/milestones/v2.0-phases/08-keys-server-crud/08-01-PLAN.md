## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP handler → Coolify REST API | All new client functions cross this boundary; payloads are Zod-validated at handler layer before reaching client |
| ofetch response → MCP internal state | Raw API responses flow into sanitizeFullProjection; PEM fields must be scrubbed before any downstream use |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-01-01 | Information Disclosure | sanitizeFullProjection reveal=true path could leak PEM | high | mitigate | Add PEM_FIELD_PATTERN hard-mask that runs even when reveal=true (D-02). Unit test asserts `private_key` and `pem` fields become '***' under reveal=true. |
| T-08-01-02 | Tampering | deleteServer default delete_volumes could be mis-sent as true | high | mitigate | Default `delete_volumes=false` in client signature; send explicit query param. Test asserts default call sends false. (D-16) |
| T-08-01-03 | Denial of Service | pollServerUntilReachable could loop forever on unreachable host | medium | mitigate | Hard 30s timeout default; returns last-seen server instead of throwing so handler can soft-succeed (D-05/D-07). |
| T-08-01-04 | Information Disclosure | COOLIFY_409 dependent UUIDs in error data | low | accept | UUIDs are non-secret resource identifiers; redactEnvelopeData already redacts string values via redactSecrets. |
</threat_model>
