<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Agent input → private_key handler | Untrusted args cross here; Zod discriminatedUnion + XOR superRefine is the gate |
| Handler → local filesystem (key_file) | Handler reads PEM from disk when `key_file` provided; path traversal risk |
| Handler → Coolify REST API | Outbound PEM material crosses this boundary (required for create/update) |
| Coolify response → agent output | sanitizeFullProjection must scrub PEM before any return path |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-02-01 | Information Disclosure | create/update response could echo PEM | critical | mitigate | Create response shape hardcoded to {uuid, name, fingerprint?} — never include private_key field. Test asserts no PEM in response. (D-02/D-03) |
| T-08-02-02 | Information Disclosure | get/list with projection:full could leak PEM | critical | mitigate | Route full projection through sanitizeFullProjection which hard-masks PEM even on reveal=true (08-01 hardening). Test asserts PEM -> '***' under reveal=true. (D-02/D-04) |
| T-08-02-03 | Tampering | key_file path traversal (../../etc/sensitive) | high | mitigate | fs.readFileSync is bounded to whatever the agent passes; document that the MCP runs with the user's filesystem privileges. No symlink resolution needed for v2.0 — agent is trusted to pass valid paths. Accept with note. |
| T-08-02-04 | Repudiation | delete without confirm could destroy a key in use | high | mitigate | Confirm gate (D-14) + COOLIFY_409 dependency check (D-15, Pitfall 2) double-gate deletes. Test asserts both gates. |
| T-08-02-05 | Elevation of Privilege | force-delete of in-use key | high | mitigate | No `force` param in schema (D-15). Test asserts schema rejects force. |
</threat_model>
