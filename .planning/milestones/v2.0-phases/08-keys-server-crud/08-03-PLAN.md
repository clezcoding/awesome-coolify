| Coolify validate response → agent output | Reachability flag drives soft-success vs pending; must not leak validation_logs secrets |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-08-03-01 | Denial of Service | validate poll hangs on dead host | high | mitigate | 30s hard timeout in pollServerUntilReachable (08-01); handler returns soft-success pending state instead of blocking (D-05). Test asserts pending status. |
| T-08-03-02 | Tampering | delete without confirm could destroy a server with children | high | mitigate | Confirm gate (D-14) + delete_preview warning (D-16). Test asserts COOLIFY_CONFIRM_REQUIRED on confirm:false. |

| T-08-03-05 | Tampering | delete_volumes default could destroy volumes | high | mitigate | Default delete_volumes=false in schema and client (D-16). Test asserts default call sends false. |
