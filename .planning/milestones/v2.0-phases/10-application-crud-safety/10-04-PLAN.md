<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Agent -> MCP delete handler | Untrusted input; Zod validates identity (uuid|name|fqdn) + confirm flag |
| MCP handler -> Coolify DELETE /applications/{uuid} | Existing bearer auth + retry + error mapping; destructive action gated by confirm |
| MCP handler -> Coolify GET /resources (for delete_preview) | Read-only; no destructive side-effect |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-10-04-01 | Tampering | delete without confirm | high | mitigate | validateDeleteConfirm throws COOLIFY_CONFIRM_REQUIRED when confirm is false; deleteApplication NOT called (SAF-01, D-18). Test asserts vi.fn() not-called. |
| T-10-04-02 | Tampering | delete with unsafe defaults | high | mitigate | Zod defaults delete_volumes/delete_configurations/docker_cleanup/delete_connected_networks to false (SAF-02, D-19). Handler passes parsed values to deleteApplication query params. Coolify API defaults these to true — MCP overrides to false at the Zod layer. |
| T-10-04-03 | Spoofing | identity resolution by name/fqdn | medium | mitigate | resolveAppMutationUuid multi-match -> COOLIFY_AMBIGUOUS_MATCH, no mutation (D-21). Existing Phase 4 pattern. |
| T-10-04-04 | Repudiation | delete audit trail | low | accept | Coolify server logs the DELETE; MCP does not own audit. Out of scope. |
| T-10-04-05 | Information Disclosure | delete_preview child resources | low | accept | Child resource list contains uuid/name/type only — no secrets. |
| T-10-04-06 | Denial of Service | accidental delete via ambiguous name | medium | mitigate | COOLIFY_AMBIGUOUS_MATCH refuses mutation on multi-match (D-21) — agent must narrow to a single UUID. |
| T-10-SC | Tampering | npm installs | high | mitigate | No new packages — reuses zod/ofetch already pinned and audited. |
</threat_model>
