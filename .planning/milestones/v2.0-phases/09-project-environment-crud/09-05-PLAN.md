# Phase 9 Plan 05: Gap Closure — initial_environment MCP Validation Envelope

**Fix live MCP rejection path for missing `initial_environment` on `project.create` — return COOLIFY_422 + recovery hints instead of SDK generic validation error.**

## Context

UAT Test 10 failed on live MCP:
- **Expected:** `COOLIFY_422` with `recoveryHints` (D-09/D-10)
- **Actual:** `Input validation error: initial_environment: Invalid input: expected string, received undefined`

Unit tests pass because they call `handleProjectAction` directly, bypassing MCP SDK `inputSchema` validation.

**Precedent:** Phase 8 Plan 05 (D-11) — `private_key.list` accepts `reveal` on schema; handler rejects `reveal:true` with COOLIFY_422.

## Goal

Live MCP `project({ action: 'create', name: '...' })` without `initial_environment` returns structured `COOLIFY_422` envelope matching unit test expectations.

## Tasks

### Task 1: Schema — optional initial_environment at MCP layer

**File:** `src/mcp/tools/project.ts`

Change `createActionSchema.initial_environment` from required `z.string().min(1)` to optional:

```typescript
initial_environment: z
  .string()
  .optional()
  .describe(
    'Initial environment name (required — ask user for production vs custom per D-09/D-10)',
  ),
```

Add comment mirroring D-11 pattern:
```typescript
// D-09/D-10: schema accepts optional initial_environment (MCP JSON Schema layer); handler rejects missing/empty.
```

**Do NOT** remove handler validation — business rule stays required, only MCP passthrough changes.

### Task 2: Handler — reject missing/empty before createProject

**File:** `src/mcp/tools/project.ts`

At start of `case 'create':`, before `createProject`:

```typescript
const initialEnv = parsed.initial_environment?.trim();
if (!initialEnv) {
  throw new CoolifyApiError({
    code: 'COOLIFY_422',
    message:
      'initial_environment is required — ask the user for production vs a custom environment name (D-09/D-10).',
    recoveryHints: INITIAL_ENV_RECOVERY_HINTS,
  });
}
```

Pass `initialEnv` (not `parsed.initial_environment`) to `ensureInitialEnvironment`.

Update `throwValidationError` / `parseProjectAction` — empty string may no longer hit Zod min(1); handler guard covers both missing and empty.
