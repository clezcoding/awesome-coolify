## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP caller → application.ts | Untrusted action + payload; Zod validates, handler resolves uuid |
| application.ts → Coolify API | Outbound HTTP via client.ts; errors redacted |
| Coolify API → MCP response | Env values cross back; masking applied before return |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-12-02-01 | Information Disclosure | env value in response | critical | mitigate | sanitizeFullProjection masks value as `***` by default; reveal:true required to unmask, and handler never auto-sets reveal (D-14/D-15). |
| T-12-02-02 | Information Disclosure | env value in error/log paths | critical | mitigate | No console.log of env payloads; all errors via mapApiError which calls redactSecrets; assert via test that no plaintext value appears in thrown error message. |
| T-12-02-03 | Elevation of Privilege | envs:delete without confirm | high | mitigate | Zod requires confirm; handler throws COOLIFY_CONFIRM_REQUIRED if confirm !== true. Test asserts the gate. |
| T-12-02-04 | Elevation of Privilege | envs:bulk-update without confirm | high | mitigate | Same confirm gate; bulk mutations are batch destructive. |
| T-12-02-05 | Tampering | envs:update by env_uuid key resolution | medium | mitigate | Fetch list, find by env_uuid, use the resolved key for the bulk PATCH — never trust caller-supplied key when env_uuid is provided (prevents key-swap attacks). |
| T-12-02-06 | Information Disclosure | reveal:true used without human preference | high | mitigate | D-15: handler surfaces an `ask_human_reveal` recovery hint in the response `data.recoveryHints` array when reveal:true is passed on ANY envs:* action (list/get/create/update/delete/bulk-update); the hint reminds the agent to ask the human first. Test asserts the hint is present. |
| T-12-02-07 | Denial of Service | envs:bulk-update with huge entries array | low | accept | Caller-bound; document a soft limit in JSDoc (e.g. 100 entries). |
</threat_model>
