<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Tool description → agent | Description text is the agent's source of truth for available actions; misleading descriptions cause agent to attempt unsupported actions or miss supported ones |
| README → human user | README is the human-facing reference; incorrect docs cause misuse (e.g. attempting cron on update, or delete without confirm) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-13-04-01 | Information Disclosure | README examples include real-looking S3 secrets | low | mitigate | Use placeholder values (e.g. `***`, `EXAMPLE_S3_SECRET`) in all README examples; never real tokens. |
| T-13-04-02 | Tampering | description lists an action the handler does not implement | medium | mitigate | Cross-reference 13-02/13-03 SUMMARYs to ensure description action lists match the actual action union. |
| T-13-04-03 | Tampering | README documents wrong frequency rules (e.g. cron accepted on update) | high | mitigate | README must explicitly state create accepts presets OR cron, update accepts presets only (Pitfall 1). Verify with grep for both `cron` and `presets` in README. |
</threat_model>
