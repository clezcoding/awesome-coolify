<assumption_delta_decision>
**Decision type:** add-alongside (NOT promote).
**Context:** D-10 locks env override over registry default for backward-compat.
**Resolution:**
- Named Instance registry is added ALONGSIDE env; env remains override of registry default.
- Primary noun: named instance (registry entry).
- env is NOT promoted above the explicit `instance` param; explicit param still wins (D-11).
- env is NOT demoted below the registry default; env stays override of registry default (D-10).
- Rationale: backward-compat — existing single-env users keep working without touching their setup; registry is purely additive.
**Implementation in resolveCredentials precedence:**
1. explicit `instance` param → registry lookup (D-11)
2. env both-set → use env (override of registry default, D-10)
3. partial env → COOLIFY_PARTIAL_ENV hard error (D-13)
4. registry default → use default entry
5. else → COOLIFY_NO_INSTANCE (D-18/D-20)
</assumption_delta_decision>
