<objective>
Wire ManifestManager.autoUpsert / autoRemove into the application, service, and database mutation handlers so every successful create/update/delete (including domain changes) keeps the manifest cache fresh. Implements D-09 (auto-hooks on all mutations) and D-11 (best-effort, non-fatal).

Purpose: Without auto-hooks the manifest drifts silently after every mutation. With auto-hooks the cache stays close to live state, reducing sync round-trips and stale-404 incidents.
Output: Three modified tool handlers calling ManifestManager.autoUpsert on create/update and ManifestManager.autoRemove on delete, with _meta.manifestWarning surfaced on best-effort failures.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Coolify API success → manifest cache | Auto-hook runs after primary success; must not leak credentials into cache |
| manifest cache failure → agent response | Best-effort warning surfaced via _meta; must not mask primary response |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-17-03-01 | Tampering | auto-hook running before primary success | high | mitigate | Hooks called only after the primary API call returns success; never before |
| T-17-03-02 | Denial of Service | auto-hook failure failing primary mutation | high | mitigate | try/catch swallows hook errors; _meta.manifestWarning only; primary response unchanged |
| T-17-03-03 | Information Disclosure | tokens passed into manifest via autoUpsert | high | mitigate | autoUpsert input shape excludes credentials; only uuid/type/name/domains/project+env UUIDs passed |
| T-17-03-04 | Tampering | concurrent auto-hooks from parallel mutations | medium | mitigate | ManifestManager.withWriteLock serializes writes (17-01) |
| T-17-03-SC | Tampering | npm installs | high | accept | No new packages — ManifestManager from 17-01 uses only zod + node:fs |

</threat_model>
